Syntaxe utilisée

Pour l'écriture de fonctions à variables booléennes, KS Boolean Expression a développé une syntaxe qui lui est propre.

Ecriture des variables:

Les variables sont écrites en lettres latines et en majuscules (A...Z). tout autre caractère ou lettre miniscule est considéré comme étant un caractère indéfini qui génère par conséquent une erreur.

Les opérateurs:

KS Boolean Expression gère six (06) opérateurs logiques: les trois opérateurs de base (NON, ET, OU) et les opérateurs universels (NAND, NOR, XOR). Les symboles qui leurs sont affectés sont les suivants:

  • ! pour le NON, ex. !A.
  • + pour le OU, ex. A+B.
  • * pour le ET, ex. A*B.
  • | pour le NAND, ex. A|B.
  • / pour le NOR, ex. A/B.
  • ^ pour le XOR, ex. A^B.

Vous pouvez aussi utiliser les parenthèses afin de regrouper les termes et indiquer la priorité des opérations.

Règles et priorités:
  • On peut s'empasser de l'opérateur (*) dans le cas d'absence de l'opérateur (!), exemple: A*B*C*D = ABCD mais A*B*C*!D est différent de ABC!D.

  • Si la formule ne contient que des opérateurs de base, i.e. le NON le ET et le OU alors la priorité est comme suit: NON--ET--OU.

  • Si la fonction ne contient que des opérateurs NAND, NOR et XOR alors la priorité est comme suit: XOR--NAND,NOR.

  • Si la fonction contient tous les opérateurs réunis alors il faut dans ce cas utiliser des parenthèses pour indiquer la priorité sinon vous risquez d'obtenir de faux résultats. Exemple: A*B|C*D--Faux, il faut écrire par exemple: (A*B)|(C*D).

  • Pour la négation d'un terme qui contient plusieurs variables il faut utiliser les parenthèses comme l'indique l'exemple suivant: A+!B --Correcte, mais A+!(BC)--Faux, il faut écrire: A+(!(BC))--Correcte.
